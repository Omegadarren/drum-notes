Section 1

What is BandNotes?

BandNotes is a tablet-first web app designed for musicians, bands, and orchestras who use digital sheet music during rehearsals and performances. It lets you open PDF sheet music, write text annotations directly on the pages (cues, dynamics, bowings, chord names, reminders — anything), and share those notes with your whole band in real time.

Everything runs inside a browser with no installation required. For the best experience, add it to your iPad Home Screen so it runs as a full-screen native-style app. All your notes are stored on your device and optionally backed up to the cloud so they follow you if you ever get a new tablet.

Core capabilities

  • Annotate PDFs — Place labelled text notes anywhere on a page with a double-tap.
  • Multi-song setlists — Swipe between songs in a setlist without going back to a menu.
  • Band Groups — Share setlists and songs across your whole band. Every member keeps their own private notes.
  • Offline-first — Once a setlist has been opened, it is cached locally and works with no internet connection.
  • Print or export — Print any session (with your annotations baked in) or save an annotated PDF.
  • Lasso mode — Select, move, align, copy, or delete multiple notes at once.
  • Metronome — Built-in click track in the viewer. Set BPM per song — the tempo syncs automatically when you swipe between songs.
Section 2

Quick Start

  1. Set your name. On the start screen, tap Change next to "Playing as" and enter your name. This is stored on-device and shown when you share notes with bandmates.
  2. Add to Home Screen (recommended). In Safari, tap the Share button ⎋ → Add to Home Screen. This makes BandNotes full-screen and keeps it available offline.
  3. Create or join a Band Group. Tap 🎸 Band Groups → Create Group (if you are the admin) or Join Group (enter the 6-character invite code from your admin).
  4. Add songs to the library (admin). Inside your group, tap 📚 Song Library → ⬆ Add Song and choose one or more PDF files.
  5. Create a setlist (admin). Back in the group, tap ✚ Setlist, give it a name, and add songs in order.
  6. Open the setlist. Tap the › arrow on any setlist row to open it in the viewer. Double-tap a page to add your first note.

Section 5

Song Library

The Song Library holds all the PDF sheet music for your group. Songs are uploaded once by an admin and are then available to all members.

Opening the library

  • Admin: Tap 📚 Song Library in the admin section of the group screen.
  • Member: Tap 📚 Song Library in the member section of the group screen.

Searching songs

An A–Z filter bar sits above the song list. Tap a letter to show only songs whose name starts with that letter. When a filter is active, a ✕ Clear button appears at the end of the bar — tap it to restore the full list.

Adding songs Admin

  1. Tap ⬆ Add Song in the library header.
  2. Tap the upload zone and select one or more PDF files from your device.
  3. Edit the name for each song (the file name is pre-filled — remove the .pdf extension if needed).
  4. Tap Upload. A progress indicator shows each upload. Songs are available to the whole group immediately.
Note — File size

Uploading large PDFs (over 20 MB) on slow connections may time out after 60 seconds. If that happens, compress the PDF first using a tool like Smallpdf or Adobe Acrobat before retrying.

Renaming a song Admin

Tap the ✏️ button on any song row, enter the new name, and tap Save. The name updates immediately across all setlists.

Replacing a song's PDF Admin

Tap the 🔄 button on any song row to swap the PDF file while keeping all annotations, BPM, play history, and arrangements intact. See Section 17 for full details.

Arranging a song Admin

Tap the ✂️ button on any song row to open the Arrangement editor. You can slice the PDF into named sections and define a custom playing order, producing a new arranged PDF that all members see. See Section 16 for full details.

Deleting a song Admin

Tap the 🗑 button on a song row and confirm. This permanently removes the PDF from storage and cannot be undone. Consider archiving any setlists that contain the song first.

Opening a song directly

Tap the › arrow or anywhere on the row (not on the action buttons) to open the song in the viewer. Your annotations for that song are loaded automatically. Tapping ‹ Back in the viewer returns you directly to the Song Library (not to the setlist screen).

Play history

If a song has been performed (via a setlist marked as performed), a 🎤 button appears on its row showing the play count (e.g. 🎤 Played 4×). Tap it to see every setlist the song was performed in with the date. This is visible to all members, not just admins. The 🎤 Played button in the song library header also opens the full group-wide Played Songs history from this screen.

Section 6

Setlists

Setlists are ordered playlists of songs from your library. Opening a setlist loads every song in sequence so you can swipe between them without interruptions during a performance.

Creating a setlist Admin

  1. On the group screen, tap ✚ Setlist.
  2. Enter a setlist name.
  3. Use the A–Z filter to quickly find songs, then tap ＋ to add them in order.
  4. Use ↑ / ↓ arrows in the selected list to reorder songs. Tap ✕ to remove a song.
  5. Tap Save Setlist ✓ when done.

Editing a setlist Admin

Tap ··· on any setlist row → Edit setlist. The same builder opens with the existing songs pre-loaded. Note: editing is only available on setlists that have not yet been marked as performed.

Opening a setlist

Tap anywhere on a setlist row to see a tooltip listing the songs it contains. Tap the › arrow to open the setlist in the viewer. BandNotes downloads all songs (or loads them from cache if offline) and your personal annotations are loaded automatically.

Archived setlists

Archived setlists are hidden under a collapsible 📦 N archived setlists section at the bottom of the list. Tap the section header to expand or collapse it. While expanded, a A→Z / Z→A sort toggle lets you quickly find setlists by name.

To restore an archived setlist to the main list, tap ··· → Restore.

Tip — Setlist song list tooltip

If an older setlist was uploaded with a plain text song list (rather than library song IDs), tapping the row shows a tooltip with the song names. Tap › to open it.

Metronome

Metronome

The metronome is a built-in click track that lives in the viewer header. The button shows the current BPM — or Set BPM (dimmed) when no tempo has been saved for the current song. Tap it to open the popover.

ControlWhat it does
− / + buttonsDecrease or increase BPM by 1. Hold down to repeat continuously.
Number inputTap the number to focus it and type a BPM value directly. Press Enter or tap elsewhere to apply.

If you navigate to a song with no saved BPM while the metronome is running, it stops automatically and a hint appears. The button label reverts to Set BPM so you know to set one.

Tip — Auto-stop

The metronome stops automatically when you tap ‹ Back to leave the viewer.

Section 8

Adding & Managing Notes

Adding a note

✌️
Double-tap (touch)
Double-tap anywhere on a page to open the note input at that position.
🖱️
Double-click (mouse)
Double-click anywhere on a page to open the note input at that position (PC / desktop).

A text input and Done / ✕ buttons appear at the tapped position. Type your note and either tap Done, press Enter, or tap anywhere else to commit. Tap ✕ or press Escape to cancel without saving.

On iOS, the keyboard Done key also commits the note.

Tip — Paste a copied note

After copying a note (see below), double-tap anywhere to open input — the copied text is automatically pasted into the field so you can quickly place it at a new position.

Editing or deleting a note

Double-tap any existing note label to open the action toast:

ButtonAction
✎ EditReopens the note input pre-filled with the existing text. Saving replaces the note at the same position.
📋 CopyCopies the note text. Double-tap elsewhere to paste it as a new note.
🗑 DeleteRemoves the note immediately.
✕Dismisses the toast without changing anything.

The toast closes automatically after 6 seconds.

Dragging a note

Press and hold a note label for 1 second until the cursor changes, then drag it to a new position on the page. Release to drop. The note's position is saved automatically.

Note style

Tap ⋯ menu → ✎ Note style to open the style panel:

  • Color — Choose from 10 colors including a pure white and a neutral gray.
  • Size — Use − / + to adjust font size from 8pt to 36pt in 2pt steps.

Tap Done to apply the style. All existing notes and any future notes use the new style.

Clearing all notes

Tap ⋯ menu → 🗑 Clear all notes. After confirming, all annotations for the current session are removed permanently.

Undo and Redo

The viewer header contains ↩ Undo and ↪ Redo buttons. Every annotation change — adding, deleting, editing, moving, pasting, aligning, or clearing notes — can be undone up to 20 steps. After undoing, the Redo button becomes available to step forward again.

Keyboard shortcuts: Ctrl+Z (or Cmd+Z on Mac) to undo; Ctrl+Shift+Z or Ctrl+Y to redo. The buttons are grayed out when there is no history.

Note

The undo history is per-session in memory only — it clears when you leave the viewer. It does not persist across browser sessions.

Section 9

Lasso Selection Mode

Lasso mode lets you select multiple notes at once to move, align, copy, or delete them as a group.

Entering lasso mode

Gesture

Two-finger double-tap anywhere on the page. Tap once (with two fingers) — pause for less than half a second — then tap again. The page background shifts slightly and the blue selection toolbar appears at the bottom.

To exit lasso mode at any time, tap the Done button in the toolbar, or perform the two-finger double-tap again.

Tip — First-time hint

The first time you open the viewer, a brief toast appears after a few seconds as a reminder about the two-finger double-tap gesture.

Selecting notes

Draw a box by pressing and dragging on empty page area. Any notes whose center falls inside the box are selected and highlighted with a blue glow.

Tap individual notes to add them to the selection one at a time.

The toolbar shows the count: e.g. 3 notes selected.

Lasso toolbar actions

ButtonWhat it does
← AlignAligns all selected notes to the left edge of the leftmost note. Requires at least 2 notes selected.
📋 CopyCopies all selected notes (text and position). A Paste button then appears in the toolbar.
📋 PasteDuplicates the copied notes, placed slightly below and to the right of the originals. Press Paste again to place another set further along. Drag them to the final position.
🗑 DeleteDeletes all selected notes after confirmation.
DoneExits lasso mode.

Group drag

With notes selected, press and hold any selected note until the drag starts, then drag all selected notes together to a new position. Release to drop. Positions are saved automatically.

Note on swipe

Horizontal swiping between songs is disabled while in lasso mode. Exit lasso mode first, then swipe to a different song.

Section 10

Sharing Notes with Your Band

Any band member can share their annotations for the currently visible song with the whole group. Other members can then view and import those notes into their own session.

Sharing your notes

  1. Open a setlist and navigate to the song you want to share.
  2. Tap ⋯ menu → 👥 Share notes.
  3. Confirm the share. Your notes for that song are uploaded to the group.

Each song can hold one set of shared notes from any one member at a time. Sharing again overwrites the previous share.

Viewing and importing shared notes

When shared notes from a bandmate are available, a purple 👥 Shared chip appears in the viewer header. Tap it to open the Shared Notes panel which shows all shared annotations grouped by song, including the sharer's name and date.

Tap Import into my notes to merge all shared notes into your own session. Imported notes get new IDs so there is no conflict with your existing notes. The chip disappears after importing.

Note — Own notes excluded

You will never see your own previously shared notes in the Shared panel — only notes shared by other members.

Section 11

Print & Export

Both options are in ⋯ menu in the viewer header. Annotations are always baked into the output.

Print (all pages)

Tap ⋯ → 🖨️ Print. BandNotes composites every page with its annotations and sends the result to the system print dialog. Each page is on its own sheet with automatic page breaks.

Save as PDF

Tap ⋯ → 📄 Print to PDF.

  • On iPad/iPhone (iOS Safari): A new tab opens with the compiled PDF. Tap the Share ⎋ button → Save to Files or AirDrop to save it.
  • On desktop Chrome/Edge: The PDF downloads to your default Downloads folder, or directly to a custom folder if one is set (see Backups & Storage).
Tip — Filename

The exported PDF is automatically named after the setlist/song (special characters replaced with underscores). For example, Sunday Service Vol 3.pdf.

By Setlist view (default)

Each setlist is shown with its date, a 🎤 tag if it was performed, and a 📦 tag if it has been archived. The songs in that setlist are listed beneath it in order.

Section 13

Offline Mode

BandNotes is designed to work at a gig even with no WiFi. Here is exactly what happens in each scenario:

ScenarioBehavior
Setlist opened before (with WiFi)All PDFs and setlist metadata are cached locally. The setlist opens from cache with full functionality — your notes are saved and load normally.
First time opening a setlist, no WiFiBandNotes tries to load from IndexedDB. Shows an error if no cache exists and explains that you need to open it once while online.
Group setlist screen, no WiFiThe setlist list is shown from local cache (last known state). Backed-up setlists show "💾" in the metadata.
Adding annotations offlineNotes save to localStorage immediately. They sync to Firestore automatically when connectivity is restored.
Sharing / importing notes offlineCloud share will fail silently with an error hint. Local notes are unaffected.
Best practice before a gig

While on WiFi before any performance, open every setlist you plan to use. This ensures all PDFs are cached and available if you lose connectivity at the venue.

Offline indicator

A yellow 📶 No WiFi banner appears at the top of the screen when BandNotes detects no internet connection. It dismisses automatically when connectivity returns.

Service Worker

BandNotes registers a Service Worker (sw.js) which caches the app shell. This means the app itself loads instantly even without network access, as long as it has been opened at least once.

Section 14

Backups & Storage

Local backup indicator (💾)

When a setlist's PDFs have been fully downloaded and cached, the setlist row shows a 💾 badge in its metadata. This confirms the setlist is available offline.

Stale backup notification

If a locally cached setlist has not been opened for 30 days, BandNotes shows a Stale Backups dialog when you open the group. You have three options:

OptionWhat it does
Keep AllMarks the backups as "keep permanently" — they will never be flagged as stale again.
Remind in 30 daysSnoozes the dialog for another 30 days.
Delete Old BackupsRemoves the stale PDF data from device storage, freeing space. The setlists themselves are still in the cloud.

Cloud sync (Firestore)

Your annotations are automatically backed up to Firestore whenever you save changes. If you get a new device or clear browser data, your notes are restored the next time you open a song from the library.

Default download folder (desktop Chrome/Edge)

On desktop Chrome or Edge, you can choose a default folder where all exported PDFs are saved automatically. Tap 📁 Default Folder in the admin section of the group screen (only visible when the File System Access API is supported).

  • Tap Choose Folder… to pick a folder. Chrome will ask for permission once.
  • All future PDF exports go straight to that folder without a save dialog.
  • Tap Clear Folder to revert to browser default Downloads.

This feature is not available on iOS/iPad — PDFs open in a new tab for sharing from there.

Section 15

Tips & Troubleshooting

Add to Home Screen (recommended)

For the best experience on iPad:

  1. Open BandNotes in Safari (other browsers don't support this).
  2. Tap the Share button ⎋ at the bottom of the screen.
  3. Scroll down and tap "Add to Home Screen".
  4. Tap Add. A BandNotes icon will appear on your Home Screen.
  5. Open BandNotes from the Home Screen — it now runs full-screen with no browser UI.

Fullscreen on Android tablets

Tap the ⛶ button in the top-right corner of the start screen. On Android Chrome the browser enters full-screen mode natively.

Multiple devices / bandmates

Each device connects with an anonymous account. Your invite code identifies the group, not the device. All band members with the code can access the same setlists while keeping entirely separate annotations.

My notes disappeared

Notes are saved to both localStorage and Firestore. If localStorage was cleared (e.g. by a browser update), they should restore from Firestore the next time you open the song while online. If Firestore is unreachable, notes load from localStorage only.

Setlist won't load offline

Open the setlist once while connected to WiFi. BandNotes will then cache all the PDFs for offline use. Make sure to do this before your rehearsal or gig if you expect limited connectivity.

Annotations appear in the wrong place after orientation change

Annotations are stored as percentages of page width/height, so they automatically reposition correctly when you rotate. If the page itself looks offset, try scrolling up/down or rotating back — BandNotes snaps the view back into alignment automatically.

The PDF looks blurry

BandNotes renders PDFs at full Retina (2× or 3×) resolution on high-DPI displays. If a PDF looks blurry, it is likely because the source PDF has low-resolution scanned images. Use a vector PDF (exported from Sibelius, Finale, Dorico, etc.) for the sharpest results.

The keyboard covers the note input on iOS

BandNotes automatically scrolls the note input into view after the keyboard appears (~350ms delay). If the keyboard still covers the input, try scrolling up slightly on the page after the keyboard opens.

Upload failed / timed out

PDF upload has a 60-second timeout per file. If uploads consistently fail, check your internet connection speed and try compressing the PDF (use Adobe Acrobat, Smallpdf.com, or Preview on Mac).

I lost my group (cleared browser data)

If you are the admin: open Band Groups — BandNotes will auto-recover groups you own from the cloud. Alternatively tap Join Group and enter your original invite code.

If you are a member: tap Join Group and ask your admin for the invite code again.

Section 16

Arranging Songs Admin

The Arrangement editor lets you slice a song's PDF into named sections — Intro, Verse, Chorus, Bridge, etc. — and then define a custom playing order to produce a brand-new arranged PDF that the whole band sees when they open that song.

Admin only

The ✂️ Arrange button is only visible to the group admin. Members see the final arranged PDF but cannot access the editor.

Opening the editor

  1. Open the Song Library.
  2. Tap the ✂️ button on the song row you want to arrange.
  3. The Arrangement editor opens with two tabs: Slice and Arrange.

Step 1 — Slice tab

The Slice tab shows all the pages of the song as thumbnails. Your job here is to draw cut lines that divide the PDF into sections.

  • Tap anywhere on a page to place a cut line at that vertical position. A horizontal line with a ▼ arrow and a name field appears at the cut.
  • The ▼ arrow means the name labels the section below that line — so "Chorus" placed at the top of the chorus music labels everything from that line down to the next cut.
  • The name field is automatically selected when you add a cut — just start typing the section name immediately.
  • The very first section (from the top of page 1 to the first cut) is always labelled Start and cannot be removed.
  • Tap ✕ on any cut line to remove it.
Tip — Name before you move

Give each section a clear name in the Slice tab before switching to the Arrange tab. Good names (e.g. "Verse 1", "Chorus", "Solo", "Outro") make it much easier to drag them into the right order.

Step 2 — Arrange tab

The Arrange tab shows your named sections as cards. The default order mirrors the original page order. Customise the arrangement using the controls on each card:

ControlWhat it does
Drag handle (☰)Press and drag to reorder the section within the arrangement.
↑ / ↓ buttonsMove the section one step up or down. Useful on small screens or when a precise drag is tricky.
＋ DuplicateInserts a copy of the section at the next position. Use this to repeat a chorus, a vamp, or any recurring passage without uploading a new PDF.
✕ RemoveRemoves the section from the arrangement entirely. The original page content is not deleted — you can re-add it from the Slice tab.

A live preview panel below the section list shows thumbnail crops of the arrangement in order so you can verify the result before saving.

Saving the arrangement

  1. When the arrangement looks correct in the preview, tap Save Arrangement.
  2. BandNotes crops each section from the original PDF and assembles them into a new PDF in the defined order.
  3. The new PDF is uploaded to cloud storage and the song is updated. All members immediately see the arranged version when they open the song.
  4. The original PDF is preserved in the background. If you open the ✂️ editor again later, it always loads the original uncut PDF so you can re-slice and re-arrange from scratch.
Note — Returning to the editor

Opening the ✂️ editor on a song that already has an arrangement will reload all previous cuts and the existing arrangement so you can refine rather than start over.

Section 17

Replacing a Song's PDF Admin

If you receive a corrected or updated version of a sheet music file, use the 🔄 Replace PDF button rather than deleting and re-uploading. Replace uploads the new file to exactly the same storage path, so every piece of existing data attached to the song is preserved.

What is preserved

DataPreserved?
All members' annotations✅ Fully intact
BPM / tempo setting✅ Fully intact
Play history & gig count✅ Fully intact
Song name✅ Fully intact
Existing arrangement (if any)⚠️ Cleared — see note below
Arrangements are cleared on replace

Because the new PDF may have different pages or different page sizes, any saved arrangement is automatically cleared when you replace the PDF. After replacing, re-open the ✂️ Arrange editor to create a fresh arrangement for the new file.

How to replace a PDF

  1. Open the Song Library.
  2. Tap the 🔄 button on the song row.
  3. Tap Choose File and select the new PDF from your device.
  4. Tap Replace. A progress bar shows the upload.
  5. Once complete, the cached copy on each device is refreshed automatically the next time members open the song.
Tip — When to use Delete vs Replace

Use Replace when you have a newer or corrected version of the same song and want to keep all existing data. Use Delete only if you are removing a song entirely from the group library.
